Hexo+GitHub建站心得

发表于 2020-09-04 更新于 2020-09-05 分类于 Technology Valine：
本文字数： 2.3k 阅读时长 ≈ 2 分钟

大体流程

准备工作

  准备 node 和 git 环境，首先，安装 NodeJS，因为 Hexo 是基于 Node.js 驱动的一款博客框架，相比起前面提到过的 Jekyll 框架更快更简洁。因为天朝网络被墙的原因尝试过安装 Jekyll 失败而放弃了。然后，安装 git，一个分布式版本控制系统，用于项目的版本控制管理。两个工具不同的平台安装方法有所不一样，可自行了解按步骤安装，这里不详述了。安装成功后打开git bash（Windowns）或者终端（Mac），下方中将统一称为命令行。在命令行中输入相应命令验证是否成功，如果成功会有相应的版本号。

  打开github，点击New repository，创建一个新仓库，仓库名必须要遵守格式：账户名.github.io，不然接下来会有很多麻烦。注意，Github 仅能使用一个同名仓库的代码托管一个静态站点，并且需要勾选Initialize this repository with a README。
在建好的仓库右侧有个settings按钮，点击它，向下拉到GitHub Pages，你会看到有个网址，访问它，你将会发现该项目已经被部署到网络上，能够通过外网来访问它，当然里面还很空什么东西都没有。 该地址就是你的博客默认地址，你也可以购买域名，将其换成你喜欢的地址。

绑定域名

  现在默认的域名还是xxx.github.io，首先你得购买一个域名，阿里云就能买。在你的域名控制台添加两条解析记录：A和CNAME，前者记录值为网站的ping，后者记录值你的GitHub仓库名。这时候你的项目根目录应该会出现一个名为CNAME的文件了。如果没有的话，打开你本地博客/source目录，新建CNAME文件，注意没有后缀。然后在里面写上你的域名，保存。最后运行hexo g、hexo d上传到github。

提前排雷

·新建的GitHub仓库名必须要遵守格式：账户名.github.io，不然接下来会有很多麻烦。

·输入自己的项目名字，后面一定要加.github.io后缀，README初始化也要勾上。

·安装Hexo时会有几个报错，一般跟系统有关，问题不大，无视它就行。
·在配置hexo主题时报错YAMLException: can not read a block mapping entry; a multiline key may not be an implicit key。报错信息提示hexo的yml配置文件，冒号后面少了空格。解决方案是到提示行将对应的空格补上即可，注意是半角。
·修改博客根目录中_config.yml文件的配置项post_asset_folder为true，并在你的文章.md的文件夹中建一个同名文件夹存放图片，引用方法为markdown语法。
·遇到问题Error: “git 不是内部或外部命令，也不是可运行的程序”。解决方法是找到git安装路径中bin的位置，将其地址复制到path后（用’ ; ‘分隔不同路径）。

心得感受

  第一感觉是独立建站并没有想象中那么难，起码不算复杂。我认为这得益于成熟的hexo相对成熟的框架和社区，有问题能找到答案。第二感觉是这种抄近路式的建站的后续工作还是挺难的。毕竟你想要的很多功能和插件需要自己配置，这期间就会遇到奇奇怪怪的问题，网上找不到答案，用着很难受。现在这个网站还有很多功能有问题，有些功能无法实现，这就是自身的水平问题，而且目前看也不是专业相关，慢慢来吧。
